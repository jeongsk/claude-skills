# Git Workflow Plugin

Git 작업 흐름을 간소화하는 커밋, 푸시 명령어 플러그인입니다.

## 개요

Git Workflow Plugin은 일반적인 git 작업을 자동화하여 컨텍스트 전환과 수동 명령 실행을 줄여줍니다. 여러 개의 git 명령어를 실행하는 대신, 하나의 슬래시 명령어로 전체 워크플로우를 처리할 수 있습니다.

## 명령어

### `/commit`

현재까지 작업한 내용을 논리적 단위별로 나눠서 커밋합니다.

**수행 작업:**
1. 현재 git 상태 분석
2. staged/unstaged 변경사항 검토
3. 최근 커밋 메시지를 참고하여 저장소의 스타일 파악
4. 작업을 논리적 단위로 분류
5. 각 단위별로 순차적으로 커밋 생성

**사용법:**
```bash
/commit
```

**사용 예시:**
```bash
# 코드 변경 후
/commit

# Claude가 수행:
# - 변경사항 분석
# - 논리적 단위로 분류
# - 각 단위별로 파일 스테이징
# - 적절한 메시지로 순차 커밋
# - 커밋 상태 표시
```

**주요 기능:**
- 저장소의 커밋 스타일에 맞는 메시지 자동 생성
- Conventional Commit 규칙 준수
- 민감한 파일(.env, credentials.json) 커밋 방지
- 각 커밋이 독립적인 의미를 가지도록 분리
- 커밋 타입: feat, fix, refactor, docs, style, test, chore

### `/commit-push`

현재까지 작업한 내용을 논리적 단위별로 나눠서 커밋하고, 원격 저장소에 푸시합니다.

**수행 작업:**
1. 현재 git 상태 분석
2. 작업을 논리적 단위로 분류
3. 각 단위별로 순차적으로 커밋 생성
4. 모든 커밋을 원격 저장소에 푸시
5. 푸시 결과 확인

**사용법:**
```bash
/commit-push
```

**사용 예시:**
```bash
# 코드 변경 후
/commit-push

# Claude가 수행:
# - 변경사항을 논리적 단위로 분류
# - 각 단위별로 커밋 생성
# - 원격 저장소에 푸시
# - 결과 확인
```

**주요 기능:**
- 논리적 단위별 순차 커밋
- 자동 푸시 처리
- 새 브랜치인 경우 upstream 자동 설정
- 푸시 실패 시 원인 파악 및 해결 안내
- 충돌, 권한 문제 자동 감지

**요구사항:**
- Git이 설치되고 구성되어 있어야 함
- 저장소에 `origin` 원격이 설정되어 있어야 함

## 설치

이 플러그인은 Claude Code 저장소에 포함되어 있습니다. Claude Code 사용 시 명령어가 자동으로 사용 가능합니다.

## 모범 사례

### `/commit` 사용하기
- 커밋 전에 변경사항 검토
- Claude가 변경사항을 분석하고 저장소의 커밋 스타일에 맞추도록 허용
- 자동 생성된 메시지를 신뢰하되, 정확성 검증
- 개발 중 일상적인 커밋에 사용

### `/commit-push` 사용하기
- 변경사항을 원격에 반영할 준비가 되었을 때 사용
- 모든 변경사항이 완료되고 테스트되었는지 확인
- 컨텍스트 전환을 최소화하고자 할 때 사용
- 푸시 전에 원격 브랜치와의 충돌 여부 확인

## 워크플로우 통합

### 빠른 커밋 워크플로우:
```bash
# 코드 작성
/commit
# 개발 계속
```

### 피처 브랜치 워크플로우:
```bash
# 여러 커밋에 걸쳐 기능 개발
/commit  # 첫 번째 커밋
# 추가 변경
/commit  # 두 번째 커밋
# 원격에 푸시할 준비 완료
/commit-push
```

## 요구사항

- Git이 설치되고 구성되어 있어야 함
- 저장소가 원격을 가진 git 저장소여야 함

## 문제 해결

### `/commit`이 빈 커밋을 생성

**문제**: 커밋할 변경사항 없음

**해결책**:
- unstaged 또는 staged 변경사항이 있는지 확인
- `git status`를 실행하여 변경사항 존재 확인

### `/commit-push`가 푸시 실패

**문제**: 푸시 명령이 실패

**해결책**:
- 저장소에 GitHub 또는 원격 저장소가 설정되어 있는지 확인
- 업스트림이 설정되지 않은 경우 자동으로 `-u` 옵션 사용
- 원격 브랜치와 충돌이 있는 경우 먼저 pull하여 병합

## 팁

- **다른 도구와 결합**: 개발 중에는 `/commit` 사용, 준비되면 `/commit-push` 사용
- **Claude가 메시지 작성하도록**: 커밋 메시지 분석은 저장소의 스타일을 학습합니다
- **푸시 전 검토**: 푸시하기 전에 항상 커밋 메시지와 변경사항을 검토하세요
- **독립적인 커밋**: 각 커밋이 독립적으로 의미를 가지고 리버트 가능하도록 유지
