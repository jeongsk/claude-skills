# Figma MCP 트러블슈팅 가이드

## 연결 문제

### MCP 서버에 연결할 수 없음

**증상**: MCP 도구 호출 시 연결 오류 발생

**해결 방법**:
1. Figma Dev Mode MCP 서버가 실행 중인지 확인
2. 서버 URL 확인 (`http://127.0.0.1:3845/mcp`)
3. 포트 충돌 여부 확인
4. Figma 데스크톱 앱이 실행 중인지 확인

**설정 확인**:
```json
{
  "figma-dev-mode-mcp-server": {
    "type": "http",
    "url": "http://127.0.0.1:3845/mcp"
  }
}
```

### 서버가 리소스를 찾을 수 없음

**증상**: "Server not found" 오류

**해결 방법**:
1. MCP 설정 파일 경로 확인
2. 서버 이름이 정확한지 확인
3. Claude Code 재시작

## 도구 사용 문제

### get_design_context가 빈 결과 반환

**증상**: 디자인 컨텍스트가 비어있음

**해결 방법**:
1. Figma에서 요소가 선택되어 있는지 확인
2. Dev Mode가 활성화되어 있는지 확인
3. 선택된 요소에 접근 권한이 있는지 확인

### get_code 결과가 불완전함

**증상**: 생성된 코드가 일부만 포함됨

**해결 방법**:
1. 복잡한 컴포넌트는 하위 요소별로 분리하여 추출
2. Auto Layout 설정 확인
3. 숨겨진 레이어 확인

### get_image가 저화질로 추출됨

**증상**: 이미지 품질이 낮음

**해결 방법**:
1. 내보내기 설정에서 해상도 확인
2. SVG 형식 사용 권장 (벡터 그래픽의 경우)
3. 레티나 대응 위해 2x 내보내기 고려

## 일반적인 문제

### Figma 파일 접근 권한 오류

**증상**: 파일에 접근할 수 없음

**해결 방법**:
1. Figma 파일의 공유 설정 확인
2. 올바른 계정으로 로그인되어 있는지 확인
3. 파일 소유자에게 권한 요청

### 대용량 파일 처리 지연

**증상**: 응답이 매우 느림

**해결 방법**:
1. 특정 페이지나 프레임만 선택
2. 불필요한 레이어 숨기기
3. 컴포넌트 인스턴스 대신 원본 사용

### 스타일 불일치

**증상**: 추출된 코드와 디자인이 다름

**해결 방법**:
1. 폰트가 시스템에 설치되어 있는지 확인
2. 색상 프로파일 확인
3. 그림자, 블러 효과의 브라우저 지원 확인

## 베스트 프랙티스

### 효율적인 작업을 위한 팁

1. **작은 단위로 작업**
   - 전체 페이지보다 컴포넌트 단위로 추출
   - 복잡한 요소는 분리하여 처리

2. **디자인 정리**
   - 명확한 레이어 네이밍
   - Auto Layout 활용
   - 컴포넌트 정의

3. **코드 후처리**
   - 생성된 코드는 기본 뼈대로 활용
   - 프로젝트 스타일 가이드에 맞게 수정
   - 접근성 및 반응형 고려

### 문제 발생 시 체크리스트

- [ ] Figma 데스크톱 앱 실행 확인
- [ ] Dev Mode 활성화 확인
- [ ] MCP 서버 실행 확인
- [ ] 올바른 요소 선택 확인
- [ ] 네트워크 연결 확인
- [ ] 권한 설정 확인
